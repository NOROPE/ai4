Tools:

Setup
    Load in the profile
        - Stored in config.json
        - Contains user preferences and settings for the tool
    Load the prevmsg.txt file
    Load the API key: stored in .env
    Create the tool API
    Create the tool Handler
    and register the ToolAPI with it - port
    Create/reuse 2 virtual audio sinks - AIOutput And AIInput
    Allow user to select sink

Shutdown:
    Save the profile
    Save the tools in the tool handler
    Stop the tool API server
    Stop the tool handler
    Reset pipewire to default state
    Save the last 30 messages from the log files to a prevmsg.txt file, delete the rest
    
Main Loop
    Google Gemini LIVE API
        - Use a live audio stream for input and output
        - Record text and transcriptions a logger which outputs both to the console and a file (the file ONLY contains the transcriptions)
            - Also handle tool requests and responses through the tools module
        - Buffer the output for 1.5 Seconds (Changable in config.json) to allow for better response times and more natural conversations
        - The output and input go to a user selected sink
        

    

tools module

Class ToolSet: (Accesses TOOLAPI)
    - A collection of tools that can be accessed by gemini
    - Each tool has a name, description, and a function that can be called
    - Can Access TOOLAPI tools

Class ToolAPI: Ran asyncronously in the background
    - Uses a shared secret token for authentication between the tool handler and the API
    - Creates the docker server and manages the tools that are registered to it
    - Checks for open ports and hosts a server which can access multiple functions on the availiable port
    - Allows running functions from the docker server
    - Several functions can be registered to the API and called by the tool handler's scripts

Future: Mixins for other tools like web browsers, file explorers, etc.

tool handler module
    - Ran within a docker container
    - Manages Tool creation and execution
    - Listens and sends requests to the ToolAPI server
    - Can run multiple tools at once and handle their responses
    - Saves tools on shutdown (docker commit) and loads them on startup
