Agentic tools restructure.

Each mixin has 1 tool.
    str -> tool -> str

This tool
    string input ->
    Per-Tool agent ->
    Thinking and executing tools ->
Tool output ->
    - Status
    - Actual useful output

How this will be managed:
    - The agent is prompted every time the MainTool is run
    - These agents are defined by individual mixins
        - Allows configuration of
            - Agent model
            - Agent location / port
        - These agents have access to the @tool_functions (exept from the agent tool call)
