### Phase 2: Logging
1. Set up dual logging вЂ” console handler (all messages) and file handler (transcriptions only)
2. Create a custom log level or filter for transcription-only output

**Test:** Log a few test messages, confirm console shows everything and file shows only transcriptions.

---

### Phase 3: PipeWire Audio Sinks
1. Implement `audio/pipewire.py` вЂ” create/reuse `AIOutput` and `AIInput` virtual sinks
2. Add sink selection prompt
3. Add PipeWire reset on shutdown

**Test:** Run, check `pw-cli list-objects` shows the sinks. Reset and confirm they're removed.

---

### Phase 4: Gemini Live API (Text Only)
1. Connect to Gemini Live API in `main.py` using asyncio
2. Send a hardcoded text message, receive and log the response
3. Add reconnection logic with retry interval from `config.json`

**Test:** Send a message, see response logged. Kill network briefly, confirm reconnection works.

---

### Phase 5: Gemini Live API (Audio)
1. Wire `sounddevice` input stream в†’ Gemini Live API
2. Wire Gemini Live API audio output в†’ `sounddevice` output stream
3. Route through the PipeWire virtual sinks
4. Log transcriptions using the logging setup from Phase 2

**Test:** Speak into mic, hear Gemini respond through the virtual sink. Check transcription log file.

---

### Phase 6: Docker Container
1. Write `Dockerfile` for the tool container
2. Implement `tool_api.py` вЂ” start/stop container using Docker SDK
3. Verify container starts on setup and stops on shutdown
4. Implement `docker commit` for saving tool state

**Test:** Run, confirm container appears in `docker ps`. Shutdown, confirm it's gone and image is saved.

---

### Phase 7: ToolAPI Server
1. Add HTTP server inside the container for tool registration and execution
2. Add shared secret token auth
3. Implement tool discovery endpoint
4. Implement tool execution endpoint

**Test:** `curl` the endpoints from host with correct/incorrect tokens. Verify auth works.

---

### Phase 8: Tools Integration
1. Implement `MixinToolAPI` вЂ” retrieve tools, manage responses
2. Implement `Tools` class вЂ” wrap tools for Gemini function calling
3. Register tools with Gemini Live API session
4. Handle tool call в†’ execute via ToolAPI в†’ return result to Gemini

**Test:** Ask Gemini to use a tool, confirm it calls the tool and receives the result.

---

### Phase 9: Shutdown & Persistence
1. Save `config.json` on shutdown
2. Save last 30 messages from log to `prevmsg.txt`
3. Load `prevmsg.txt` on startup as conversation context
4. Full graceful shutdown sequence

**Test:** Run a conversation, shut down, restart. Confirm previous context is loaded and PipeWire is clean.

---

### Phase 10: Polish
1. Error handling throughout
2. Edge cases (no mic, Docker not running, API quota exceeded)
3. End-to-end test of full lifecycle

---

### Order Rationale
Each phase **builds on the previous** and is independently testable. Audio and API are validated before introducing Docker complexity.
